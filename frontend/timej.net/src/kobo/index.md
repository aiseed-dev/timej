# AISeed工房（pykobo） — Pythonで動く日本語ワープロ

AISeed工房は、**テキストで書き、組版で確かめ、紙とデータに出す**ための
日本語エディタです。昔の日本語ワープロがやっていたことを、いまの道具
（AI・OSS・Python）でやり直します。青空文庫の注記形式やAsciiDocで執筆でき、
縦書きの組版プレビューやEPUB/PDFの書き出しまでを一つのアプリで行います。

Flet製なので、**画面も業務ロジックも同じPython・同じプロセス**です。
変換ライブラリ（[文庫/pybunko](../bunko/)）や組版ライブラリ（pywashi）を
そのまま `import` して直接呼べます——サーバもAPIも、言語の切り替えも要りません。

## なぜワープロを作り直すのか

日本語の文章を「書いて・組んで・配る」道具は、長らく商用ワープロと
オフィスソフトが担ってきました。けれど、縦書き・ルビ・傍点といった
日本語固有の組版は、いまも多くのツールで十分に扱えません。一方で、
組版はWeb標準（CSS）とヘッドレスブラウザで美しく出せるようになり、
コードはAIが書けるようになりました。

だから、閉じた製品に合わせるのではなく、**開いた標準（テキスト・Markdown・
AsciiDoc・CSS組版）の上に、自分の手に馴染むワープロを生やす**。それが工房です。

## できること

- **執筆** — 青空文庫注記／AsciiDocで書く。語を選んでルビ・傍点、字下げ・
  見出し・挿絵、表（xlsxから挿入）。保存は構造化データ（.pykobo）
- **組版で確かめる** — 縦書きの組版プレビュー（pywashiに委譲）。印刷は
  ブラウザの印刷ダイアログ、印刷用PDFも
- **出す** — 青空注記テキスト（UTF-8／提出用Shift_JIS）・JSON・EPUB・
  公式XHTML再現
- **外部エディタ連携** — 使い慣れたエディタ（VSCode・OnlyOffice等）で開き、
  保存を検知して取り込む。特定製品を名指ししない設計
- **AIの活用** — 底本の写真から青空注記への書き起こし（ローカルVLM）、
  Claudeによる校正の下読み

## 三つの層で分ける

工房は「薄いUI」に徹し、重い仕事は関連コンポーネントに委ねます。
二重実装を避けるための設計です。

| 層 | 担当 |
|---|---|
| **入力・表示（工房 / pykobo）** | 書く・見る・出すの操作。Flet |
| **青空注記の解析・変換** | [文庫 / pybunko](../bunko/) |
| **日本語組版（縦書き・原稿用紙・PDF）** | pywashi（Markdown/AsciiDoc入力） |
| **AsciiDocの解釈** | pyasciidoc（CJKフレンドリー） |

工作員（入力・校正）向けの作業台として生まれ、いまは書き手のための
ワープロへ育っています。サーバは建てません。
