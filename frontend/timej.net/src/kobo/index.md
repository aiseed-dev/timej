# AISeed工房（pykobo） — 日本語の文書をAIと作る作法

これは [AIネイティブな仕事の作法](https://aiseed.dev/ai-native-ways/)（構造）の、
**文書・デザイン**の実践です。工房は「テキストで書き、組版で確かめ、紙と
データに出す」ための日本語エディタ（Flet製）。昔の日本語ワープロがやっていた
ことを、いまの道具（AI・OSS・Python）でやり直します。

## 作法 — 中身と外枠を分ける

この作法の核は、**残すべき中身**と、**使い捨ての外枠**を分けることです。

- **中身** … 外字まで解決した構造化テキスト（Document）。これが一次表現で、
  ずっと残す
- **外枠** … 縦書きの組版・HTML・PDF・EPUB。中身からいくらでも生成できる、
  派生物

だから工房は組版を自分で抱え込まず、[文庫(pybunko)](../bunko/)で構造化し、
縦書き組版は pywashi に委ねます。UIも変換も同じPython・同じプロセスなので、
これらのライブラリを `import` して直接呼べます——サーバもAPIも要りません。

## 作法 — 人が判断し、AIが労働する

工房での文書仕事は、**人は「何を・どう」を決め、AIは手間を担う**という
分担で流れます。

1. **書く** — 青空文庫注記／AsciiDocで執筆。語を選んでルビ・傍点、見出し・
   字下げ・表（xlsxから挿入）
2. **（底本があれば）撮る → AIが書き起こす** — 底本の写真を、ローカルのVLMが
   青空注記の下書きに起こす（スキャナ＋OCR＋手修正の置き換え）
3. **機械チェック** — 未対応注記・未解決の外字を機械が洗い出す
4. **AIが下読み** — Claudeが校正の疑いを挙げる。**採否は人が底本で判断**する
   （AIに最終決定をさせない＝サンドボックスの作法）
5. **組版で確かめる** — 縦書きプレビューで仕上がりを見る
6. **出す** — 青空注記テキスト・JSON・EPUB・公式XHTML・印刷用PDF

AIが起こし・洗い出し・下読みという労働を引き受け、人は底本と付き合わせて
判断する。この境界を守るのが、速さと正しさを両立させる勘所です。

## 作法 — 道具は開いた標準の上に生やす

縦書き・ルビ・傍点は、閉じた製品ではいまも十分に扱えません。けれど組版は
Web標準（CSS）とヘッドレスブラウザで美しく出せ、コードはAIが書ける。
だから閉じた製品に合わせず、**開いた標準（テキスト・AsciiDoc・CSS組版）の
上に、自分の手に馴染むワープロを生やす**。これが工房の立ち方です。

外部エディタ連携もこの作法の一部で、`KOBO_EDITOR` に好きなエディタ
（VSCode・OnlyOffice等）を指すと、そこで書いて保存を工房が取り込みます。
特定製品を名指ししません。
