レシピ#9-1: AIと創るデザインシステム

優れたアプリは、一貫したデザイン言語を持っています。色の使い方、文字の大きさ、ボタンの形... これらのデザイン原則を体系的にまとめたものを「デザインシステム」と呼びます。

通常、デザインシステムの構築は専門のUI/UXデザイナーの領域ですが、AIを「クリエイティブな相談相手」として活用することで、開発者でも体系的なデザインの土台を築くことができます。

このレシピでは、コーディングを始める前に、AIとの対話を通じてアプリのデザインシステムの核となる要素（カラーパレット、タイポグラフィなど）を決定していくプロセスを学びます。

1. AIと一緒にデザインコンセプトを創る

2. カラーパレットの生成

デザインコンセプトが決まったら、次はそのコンセプトを体現する色の組み合わせ（カラーパレット）をAIに生成させます。

🤖 AI活用プロンプト (カラーパレット生成)

ありがとう。「手書きのような温かみと親密さ」というコンセプトが気に入りました。

このコンセプトに基づいた、FlutterのMaterial Design 3に準拠するカラーパレットを提案してください。

要件: - プライマリカラー (Primary) - セカンダリカラー (Secondary) - ターシャリカラー (Tertiary) - エラーカラー (Error) - 背景色 (Background) - テキストやアイコン用の色 (On Primary, On Backgroundなど)

それぞれの色のHEXコードと、その色がコンセプトの中でどのような役割を果たすかの簡単な説明を添えてください。例えば、「温かみのあるクリーム色を背景に、安心感を与える落ち着いた緑をプライマリカラーとする」といった説明をお願いします。

このプロンプトにより、あなたはただの色のリストではなく、色の意味まで含んだ、ストーリーのあるカラーパレットを手に入れることができます。

3. タイポグラフィ（フォント）の選定：現実的なベストプラクティス

文字のスタイル（タイポグラフィ）は、アプリの印象と、何よりも**「読みやすさ」**を決定づける重要な要素です。

しかし、日本語フォントを扱う際には、デザインの自由度と、アプリのパフォーマンスやファイルサイズという、現実的な制約との間で、賢明なバランスを取る必要があります。

日本語フォントの課題：「重さ」

アルファベットが数十文字であるのに対し、日本語は、ひらがな、カタカナ、そして数千もの漢字を含みます。 そのため、カスタムフォント（Google Fontsなど）をアプリに導入しようとすると、フォントファイルだけでアプリのサイズが数十MBも増加してしまう、という深刻な問題が発生します。これは、ユーザーのダウンロード体験を著しく損なう原因となります。

特にFlutter Webでは、フォント読み込みの遅延や、初期レンダリングのちらつき（FOIT: Flash of Invisible Text）といったUX上の問題も起こりやすく、慎重な対応が必要です。

結論：システムフォントを最大限に活用する

幸い、現代のスマートフォンOS（iOS, Android）には、**非常に高品質で、可読性の高い、美しい日本語の「システムフォント」**が標準で搭載されています。

特別なブランディング上の理由がない限り、この優れたシステムフォントをそのまま利用することが、パフォーマンスとユーザー体験の両方を満たす、最も推奨されるベストプラクティスです。

私たちの役割は、フォントの種類を選ぶことではなく、システムフォントを使いこなし、文字の「見せ方」をデザインすることです。

たとえば、文字の太さ（fontWeight）や行間（height）を調整するだけでも、印象や読みやすさは大きく変わります。 また、英数字のみを装飾的なフォントに変えるといったアプローチも可能ですが、日本語フォント全体をカスタムにする必要は基本的にありません。

🤖 AI活用プロンプト (タイポグラフィ・スタイルの定義)

次に、タイポグラフィのスタイルを決めます。 私たちのアプリのコンセプトは**「信頼できる、冒険者のためのフィールドノート」**です。

Flutterの標準のシステムフォントを利用することを前提として、このコンセプトを表現するためのTextThemeのスタイルを提案してください。

要件:

  1. ThemeDataのtextThemeプロパティに設定できる、具体的なコードを生成してください。
  2. 見出し (Headline) / タイトル (Title): 信頼感と読みやすさを両立させるため、少し太め (fontWeight) に設定してください。
  3. 本文 (Body): 長時間読んでも疲れないように、適切な文字サイズ (fontSize) と行の高さ (height) を設定してください。
  4. それぞれのスタイルについて、なぜその設定がコンセプトに適しているのか、簡単な説明を加えてください。

AIからの提案（期待されるアウトプット例）

このプロンプトにより、AIは、フォントファミリーを変更するのではなく、既存のシステムフォントの「スタイル」を調整する、という、より専門的で、実践的なコードを生成します。

// app_theme.dart の ThemeData 内に設定する TextTheme

textTheme: ThemeData.light().textTheme.copyWith(
  // --- 見出し ---
  headlineMedium: ThemeData.light().textTheme.headlineMedium?.copyWith(
    fontWeight: FontWeight.bold, // 太字にすることで、情報の重要度と信頼感を表現
    letterSpacing: 1.2, // 文字間を少し広げ、ゆったりとした印象に（※環境によって視覚効果が異なるため注意）
  ),

  // --- 各画面のタイトル ---
  titleLarge: ThemeData.light().textTheme.titleLarge?.copyWith(
    fontWeight: FontWeight.w700, // Boldより一段階太くし、力強さを演出
  ),

  // --- 本文 ---
  bodyLarge: ThemeData.light().textTheme.bodyLarge?.copyWith(
    height: 1.5, // 行の高さを1.5倍に設定。長文でも読みやすく、目の負担を軽減
  ),
  bodyMedium: ThemeData.light().textTheme.bodyMedium?.copyWith(
    height: 1.5,
  ),
)

このアプローチにより、私たちは、アプリのパフォーマンスを一切犠牲にすることなく、デザインコンセプトに沿った、美しく、読みやすいタイポグラフィを、アプリ全体で一貫して実現することができます。

また、Flutterアプリの初期段階では、「フォントそのもの」ではなく「使い方（階層・余白・太さ）」を整えることがデザインの要となります。

フォントの指定は最小限に抑え、文字の見せ方の工夫に集中するという考え方は、今後あらゆるUI/UX設計の基盤となるでしょう。

4. コンポーネントデザインの原則

最後に、ボタンやカードといった基本的なUIコンポーネントが、どのようなデザイン原則に従うべきかを定義します。

🤖 AI活用プロンプト (コンポーネントデザイン定義)

デザインシステムの仕上げとして、主要なコンポーネントのデザイン原則を定義します。 これまでのコンセプト（温かみ、親密さ）と、決定したカラーパレット、タイポグラフィを踏まえて、以下のコンポーネントのデザインについて、具体的なスタイルを提案してください。

  • ボタン (ElevatedButton):
  • 角の丸み (borderRadius) はどのくらいが良いか？(例: BorderRadius.circular(20) のように具体的に)
  • 影 (elevation) は強くすべきか、控えめにすべきか？
  • カード (Card):
  • カードの背景色は？
  • 影の強さや広がりは？
  • テキスト入力欄 (TextField):
  • 枠線はどのようにデザインすべきか？（下線のみ、全体を囲む、など）

これらのスタイルをThemeDataのelevatedButtonTheme, cardTheme, inputDecorationThemeで設定するための、具体的なコード例も提示してください。

このように、各コンポーネントのデザイン原則をAIと一緒に定義することで、アプリ全体に一貫性のある見た目と操作性を与えることができます。

👉 デザインが固まったら、「このテーマを lib/theme/app_theme.dart として作成し、lib/main.dart の MaterialApp に組み込んで」とエージェントに依頼しましょう。 Claude Code / Claude Codeなら、ファイルの作成からmain.dartへの組み込みまでを直接行ってくれます。 あなたのプロジェクト構成に合わせて、適切な形で反映してくれるはずです。

このレシピを通じて、あなたはAIとの対話だけで、アプリ全体の一貫性を担保するThemeDataの雛形を完成させることができます。これは、後のコーディングフェーズにおいて、驚異的な生産性の向上をもたらす、最も価値のある「準備」です。
